FAQ

Common questions

Never. Everything is stored locally in your browser using Chrome's local storage API. No server ever receives your screenshots or guide data. You can verify this yourself in the open-source code.
Chingu needs activeTab permission (to capture screenshots when you're recording) and tabs permission (so the floating REC bar persists when you switch tabs). It does not request access to cookies, passwords, browsing history, or background data.
Yes — any standard web page in Chrome. There are a few exceptions: Chrome's internal pages (chrome://), the Chrome Web Store itself, and other extension pages. Chrome blocks all extensions from injecting into those, not just Chingu.
No catch. Chingu is open source under the MIT license and built as a free tool. There is no paid tier, and none is planned. If that ever changes, the existing open-source code stays free forever — you can fork it and run your own version.
Three main differences: (1) completely free with no account required, (2) stores everything locally instead of in the cloud, (3) open source. Scribe and Tango both require accounts, store guides on their servers, and gate features like PDF export behind paid tiers. Chingu's PDF export is free.
Yes. Every step's text is fully editable in the guide editor. You can also reorder steps by dragging, delete steps, group steps into named section chapters (A, B, C…), and add callout notes — info tips or warnings — to any step.
Yes. Set a brand accent color in settings — it's used across the PDF cover page, section dividers, step badges, and footer. No "Made with Chingu" watermarks unless you choose to keep them. (And yes, that's also free.)
Chrome and Edge are supported (both run on Chromium). Firefox and Safari aren't supported yet — they use different extension APIs and would need a separate build. It's on the roadmap; track it on GitHub.
